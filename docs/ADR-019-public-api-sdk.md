# ADR-019: Public OpenAPI + Official SDKs

## Status: Accepted

## Context

Before FASE 13.4:

- **Swagger UI** was live at `/v1/docs` (via `@fastify/swagger-ui`),
  but the generated spec was ephemeral — never checked into the repo,
  never versioned, never surfaced to external integrators.
- **No SDK.** Customers wiring GovAI into their own apps wrote raw
  `fetch` / `requests.get` calls against hand-transcribed route docs.
  Every API change broke these integrations silently.
- **No documented rate-limit contract.** The platform already emitted
  a `Retry-After` header on 429s, but consumers had no way to
  implement proactive backoff — the `X-RateLimit-*` triad was missing.
- **No developer portal.** "How do I integrate?" routinely required a
  Slack thread with platform engineering.

Competitive landscape (short):

| | CredoAI | Holistic AI | GovAI pre-13.4 | GovAI post-13.4 |
|---|---|---|---|---|
| Public OpenAPI, versioned in-repo | ✓ | partial | ✗ | **✓** |
| TypeScript SDK | ✓ | ✗ | ✗ | **✓** |
| Python SDK | ✓ | ✓ | ✗ | **✓** |
| Rate-limit headers standard triad | ✓ | ✗ | partial | **✓** |

## Decision

### Versioned OpenAPI spec

`docs/api/openapi.{json,yaml}` — produced by
`scripts/export-openapi.ts` which boots the live Fastify instance
with `GOVAI_SKIP_LISTEN=true`, calls `fastify.ready()` + `swagger()`,
and writes both JSON and YAML representations.

CI drift guard (`scripts/check-openapi-drift.ts`, exposed as
`npm run openapi:check`) boots the same instance and compares
against the committed `openapi.json`. If a developer adds or changes
a route without regenerating the spec, CI fails with a summary of
added/removed paths.

**Why JSON for the canonical compare:** YAML serialization has
nondeterministic spacing/quoting across `js-yaml` versions; JSON
is bit-stable with `JSON.stringify(spec, null, 2)`. The YAML file is
regenerated alongside each JSON export, so both stay in sync.

### TypeScript SDK — `sdk/typescript/` → `@govai/sdk`

Thin wrapper over [`openapi-fetch`](https://openapi-ts.dev/openapi-fetch/)
(~2 kB runtime). Types are generated from `docs/api/openapi.yaml` by
`openapi-typescript`. Public surface:

```ts
import { createGovAIClient } from '@govai/sdk';

const client = createGovAIClient({
    baseUrl: 'https://api.yourcompany.com',
    apiKey: process.env.GOVAI_API_KEY!,
    orgId: process.env.GOVAI_ORG_ID!,
});
const { data, error } = await client.GET('/v1/admin/assistants');
```

Every path / param / request body / response is typed. Invalid
routes or misnamed params fail at compile time.

Bundled middleware:

- **`autoBackoff: true`** (default) reads `X-RateLimit-Remaining`
  and pauses the next request when <5 % of the bucket remains. On a
  429, honors `Retry-After` with a 60 s cap.
- Passing `autoBackoff: false` disables this for consumers who want
  custom strategies (token bucket, circuit breaker, etc.).

### Python SDK — `sdk/python/` → `govai-sdk`

Generated by [`openapi-python-client`](https://github.com/openapi-generators/openapi-python-client),
configured via `sdk/python/config.yaml`. Produces:

- `govai_sdk.AuthenticatedClient` — httpx-backed client with
  Bearer-token auth.
- `govai_sdk.api.*` — one module per tag (assistants, policies,
  bias-assessments, icp-certificates, …) with `sync` / `asyncio`
  call helpers.
- `govai_sdk.models.*` — attrs-based typed request / response models.

The generator is a **dev-time** dependency. Generated files are
committed so end users only need `pip install govai-sdk` — no
build-step on their side.

### Rate-limit headers

Extended `@fastify/rate-limit`'s `addHeaders` to emit the canonical
triad:

```
X-RateLimit-Limit:     1000
X-RateLimit-Remaining: 987
X-RateLimit-Reset:     1713357600
Retry-After:           42       (on 429 only)
```

Documented in `docs/api/RATE_LIMITS.md` — per-caller buckets,
route-level overrides, observability surface.

### Developer Portal — `/developers`

Static page in the admin UI with:

- Base URL + auth format
- Link to live Swagger UI
- Copy-able code snippets in TS / Python / cURL
- Rate-limit table
- Links to the spec file on GitHub + SDK source

## Trade-offs

- **Spec ships as both JSON and YAML.** Slight duplication (the YAML
  is derived from the JSON every export). Kept because YAML is easier
  to hand-review in PRs and some generators prefer it. Drift check
  only compares the JSON.
- **Server.ts still auto-calls `start()` at module load.** The
  guarding via `GOVAI_SKIP_LISTEN=true` is a 3-line change that
  preserves the existing dev/prod entry point without a refactor to
  `buildApp()`. A future `buildApp()` split can happen without
  breaking consumers.
- **Python SDK generator is not a runtime dependency**, but the
  first-time setup still needs `pip install openapi-python-client`.
  The README captures the recipe; the generated files are
  re-committed after each spec change by the team member who edits
  the spec. Acceptable for a repo where the TypeScript generator is
  the primary path.
- **The exporter initializes workers** (BullMQ consumers, shield
  schedule, etc.) because they run at module top level. They do no
  harm for the short export/check runs — connections are released
  via `fastify.close()`. Splitting them out would require a real
  `buildApp()` refactor; left as a follow-up.

## Alternatives considered

- **OpenAPI Generator (Java CLI)** — more generators per language,
  but the output is heavier and the Java runtime dependency is
  annoying on dev laptops. `openapi-typescript` + `openapi-python-client`
  give 95 % of the value with 10 % of the friction.
- **Hand-written SDKs** — previous state. Immediately diverged from
  the API within a release. Generated SDKs are the only stable path.
- **Hosted spec only (no repo file)** — failed the drift-check goal:
  the spec must be committed so PRs have a reviewable diff.
- **Public the entire OpenAPI UI without auth** — considered. Current
  `/v1/docs` stays on the live API domain (Swagger UI). A follow-up
  can host a static copy on a CDN if marketing wants a polished
  "API reference" page.

## Consequences

- External developers can integrate with the platform using a
  typed SDK in their language of choice.
- Spec drift is caught at PR time, not by angry customers.
- Rate-limit semantics are now part of the public API contract —
  any future change requires bumping the spec version.
- RFP questions "Do you have a public API / SDK?" are answerable
  with live links.
- Follow-ups: publish both SDKs to npm / PyPI with a release workflow,
  add request-correlation IDs (`X-Trace-Id`) to the spec as a
  recommended practice, extend the drift check to the request/response
  schemas (not just path set).
